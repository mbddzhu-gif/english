# 英语学习应用 - 开发计划

## 项目概述

开发一个功能完整的英语学习应用，核心流程：**拍照/上传图片 → 识别主体 → 显示英语单词和短语 → 跟读评分 → 生成场景图片 → 创建场景对话**。

## 技术栈

- **前端**：纯 HTML5 + CSS3 + Vanilla JavaScript (ES6+)，无框架依赖
- **AI API**：MiniMax API（图片理解、文本生成、语音合成、图片生成）
- **语音识别**：Web Speech API (`SpeechRecognition`)
- **音频录制**：`MediaRecorder API`
- **摄像头**：`MediaDevices API`
- **架构模式**：SPA 单页应用 + Class 模块化（参照翻译工具项目）

## MiniMax API 使用方案

| 功能 | API 端点 | 模型 | 说明 |
|------|---------|------|------|
| 图片理解（主体识别） | `/v1/text/chatcompletion_v2` | `MiniMax-Text-01` | 多模态消息，传入图片+识别提示词 |
| 短语生成 | `/v1/text/chatcompletion_v2` | `MiniMax-M2.7` | 基于识别主体生成英语短语 |
| 发音评分 | `/v1/text/chatcompletion_v2` | `MiniMax-M2.7` | AI评估用户发音文本 vs 参考文本 |
| 语音合成（同步） | `/v1/t2a_v2` | `speech-2.8-hd` | 短语跟读示范音频 |
| 场景图片生成 | `/v1/image_generation` | `image-01` | 基于主体生成场景图片 |
| 对话生成 | `/v1/text/chatcompletion_v2` | `MiniMax-M2.7` | 基于场景生成英语对话 |

## 项目文件结构

```
微课堂2/
├── index.html              # 主入口页面（SPA）
├── css/
│   └── style.css           # 全局样式
└── js/
    ├── utils.js            # 工具函数（Toast、JSON修复等）
    ├── api.js              # MiniMax API 封装
    ├── speech.js           # 语音合成与识别管理
    ├── scoring.js          # 发音评分引擎
    ├── input.js            # 图片上传/相机拍摄模块
    ├── phrase.js           # 短语生成与管理
    ├── scene.js            # 场景图片生成与对话创建
    └── app.js              # 应用主控制器
```

## 页面流程设计

```
首页(上传图片) → 识别结果页(单词+短语) → 跟读评分页 → 场景对话页
```

SPA 页面 Section 划分：
1. `page-home` - 首页：图片上传/相机拍摄入口
2. `page-result` - 识别结果页：显示主体单词+英语短语
3. `page-practice` - 跟读评分页：录音+多维度评分
4. `page-scene` - 场景对话页：场景图片+英语对话
5. `page-loading` - 加载过渡页
6. `page-settings` - API设置页

## 阶段一：图像上传与主体识别功能开发

### 1.1 项目基础搭建
- 创建 `index.html` 主页面，定义所有 section
- 创建 `css/style.css`，建立设计系统（CSS变量、基础布局、组件样式）
- 创建 `js/utils.js`，实现 Toast 提示、JSON 修复等工具函数

### 1.2 MiniMax API 封装 (`js/api.js`)
- 基于 `chatCompletionWithImage` 实现图片理解功能
- 识别提示词设计：要求AI返回JSON格式，包含主体英文名、中文名、分类
- 封装 `identifyObject(imageBase64)` 方法
- API配置管理（localStorage存储API Key）

### 1.3 图片上传/相机模块 (`js/input.js`)
- 从相册选择图片（`<input type="file" accept="image/*">`）
- 相机拍摄（`getUserMedia` + Canvas截图）
- 图片压缩与Base64转换
- 拖拽上传支持
- 图片预览功能

### 1.4 识别结果展示界面
- 显示上传的图片缩略图
- 显示识别到的主体物体名称（英文+中文）
- 显示主体分类（动物/食物/物品/自然等）
- 加载状态与错误处理

### 1.5 阶段一测试
- 图片上传功能测试（相册选择、相机拍摄）
- 图片识别准确性测试
- 识别结果展示完整性测试
- 错误场景测试（无网络、API错误、无法识别的图片）

## 阶段二：短语生成与跟读评分功能开发

### 2.1 短语生成模块 (`js/phrase.js`)
- 基于识别主体，调用 `chatCompletion` 生成英语短语
- 提示词设计：生成3-5个包含该主体的实用英语短语，返回JSON
- 短语内容包含：英语短语、中文翻译、音标
- 短语难度分级（简单/中等）

### 2.2 语音合成模块 (`js/speech.js`)
- MiniMax TTS API 封装（同步语音合成）
- 短语示范朗读功能
- 逐词高亮播放
- Web Speech API 降级方案
- 播放控制（播放/暂停/停止/语速调节）

### 2.3 录音功能
- MediaRecorder API 录音实现
- 录音状态管理（准备/录音中/完成）
- 录音波形可视化（Canvas + AnalyserNode）
- 录音时长限制（最长30秒）

### 2.4 语音识别
- Web Speech API 语音识别
- 识别用户跟读内容
- 实时显示识别文本

### 2.5 发音评分引擎 (`js/scoring.js`)
- **发音准确度**：编辑距离算法，对比识别文本与参考短语
- **流利度**：基于录音时长与词数计算语速评分
- **语调评分**：基于音频波形特征分析（音高变化、停顿模式）
- **综合评分**：加权计算总分
- 评分结果可视化：环形进度图 + 各维度柱状图
- 改进建议生成

### 2.6 评分反馈界面
- 综合评分环形图展示
- 准确度/流利度/语调三维评分展示
- 逐词对比展示（正确/错误/遗漏标记）
- 改进建议文字提示
- 重新跟读按钮

### 2.7 阶段二测试
- 短语生成质量测试
- 语音合成播放测试
- 录音功能测试
- 评分算法准确性测试
- 评分界面展示测试

## 阶段三：场景图片生成与对话创建功能开发

### 3.1 场景图片生成 (`js/scene.js`)
- 基于识别主体，调用 MiniMax 图片生成API
- Prompt设计：生成包含该主体的生活场景图片
- 图片加载状态展示
- 图片生成失败重试机制

### 3.2 场景对话生成
- 基于场景图片+主体，调用 `chatCompletion` 生成对话
- 对话格式：2-3轮简单英语对话
- 每轮对话包含：说话人、英语内容、中文翻译
- 对话内容适合英语学习者，难度适中

### 3.3 对话展示界面
- 场景图片展示
- 对话气泡式展示（区分不同说话人）
- 对话音频播放（TTS逐句朗读）
- 当前播放句子高亮

### 3.4 场景对话练习
- 用户角色扮演：选择一个角色进行对话
- 语音输入回答
- AI评估用户回答
- 对话进度指示

### 3.5 整体流程优化
- 各阶段过渡动画
- 加载状态优化（进度条+提示文字）
- 错误处理与重试机制
- 返回/重做功能
- 响应式布局优化

### 3.6 阶段三测试
- 场景图片生成测试
- 对话生成质量测试
- 对话练习流程测试
- 端到端流程测试
- 性能与用户体验测试

## UI 设计规范

- **主色调**：`#4F46E5`（靛蓝）- 学习/知识感
- **成功色**：`#10B981`（绿色）- 正确/高分
- **警告色**：`#F59E0B`（橙色）- 提示/中等分
- **错误色**：`#EF4444`（红色）- 错误/低分
- **圆角**：12px 统一
- **布局**：移动端优先，max-width: 480px 居中
- **字体**：系统默认字体栈
- **动画**：all 0.3s ease 过渡

## 开发顺序

按阶段顺序执行，每个阶段完成后进行全面测试，确认无误后进入下一阶段。阶段一为基础，必须稳定后再开发阶段二；阶段二为核心交互，必须流畅后再开发阶段三。
